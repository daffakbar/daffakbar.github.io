Conflict Repository

Penyebab terjadinya conflict pada git.


Conflict pada git terjadi karena adanya mengubah, menghapus, menambahkan baris baru pada file yg sama.
seperti contoh berikut :
dari branch master kita menambahkan code pada baris 5
lalu dari branch "test" kita tambahkan juga code pada baris 5
maka saat akan di merge akan terjadi conflict berikut.

Cara menghindari conflict.


  • sebelum melakukan coding lebih baik melakukan merge ke master agar update jika adanya perubahan baru
  • melakukan komunikasi pembagian file/tugas dengan team

Apa yang harus dilakukan apabila terjadi conflict.



saat terjadi conflict maka terdapat beberapa pilihan berikut :
  • Accept Current Change
  • Accept Incoming Change
  • Accept Both Change
  • Compare Change
jika kalian memlih Accept Current Change maka kalian akan memakai code pada branch test.

jika kalian memlih Accept Incoming Change maka kalian akan memakai code pada branch master.

jika kalian memlih Accept Both Change maka kalian akan memakai keseluruhan code tanpa menghapus salah satunya.

jika kalian memlih Compare Change maka kalian akan diperlihatkan code dari branch test dan master

jangan lupa setelah kalian selesai memilih lakukan ini
