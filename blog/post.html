Git dan GitHub

Version control system.


merupakan tools perangkat lunak yang membantu software team dalam mengelola perubahan source code dari waktu ke waktu, berguna bekerja berkolaborasi dengan lebih baik, mengethaui siapa dan kapan sebuah perubahan terjadi, memungkinkan untuk kembali ke keadaan sebelum perubahan

Git.


merupakan open source version control system yang digunakan oleh para developer untuk mengatur segala sesuatu pengembangan software secara bersama - sama dari project kecil sampai project besar dengan mengandalkan kecepatan dan efisiensi.

berikut merupakan perintah dasar git yang perlu kalian tau :

  • "Git init" berguna untuk menginisialisasi repository git pada project, kemudian akan membuat folder .git
  • "Git clone" berguna untuk menduplikasi repository
  • "Git status" berguna untuk mengetahui status perubahan pada work directory lokal
  • "Git add " berguna untuk mengubah status file / direktori menjadi staged
  • "Git commit" berguna untuk memasukkan data snapshot ke dalam database git
  • "Git push" berguna untuk mengirimkan perubahan file setelah di commit ke remote repository.
  • "Git branch" berguna melihat seluruh branch yang ada pada repository
  • "Git checkout" berguna menukar branch yang aktif dengan branchyang dipilih
  • "GIt merge" berguna untuk menggabungkan branch yang aktif dan branch yang dipilih

GitHub.


GitHub merupakan website dengan Version Control System sebagai penyedia layanan untuk penyimpanan repositori Git dan Github adalah pusat kolaborasi untuk pengembangan software. Sebagian besar dari semua repositori Git tersimpan di GitHub.

berikut merupakan fungsi dari GitHub :

  • sebagai penyimpanan repository
  • sebagai tempat untuk berkolaborasi dalam pengerjaan sebuah software
  • sebagai tempat social media sesama programmer, dapat memfollow programmer lain
  • Watch/dapat mengawasi repository tertentu
